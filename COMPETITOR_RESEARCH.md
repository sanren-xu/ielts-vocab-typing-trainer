# IELTS Typing Learner 竞品调研

## 1. 调研目的

本次调研不是为了模仿所有背单词产品，而是为了确认 IELTS Typing Learner 下一阶段应该如何取舍：

- 学习 Qwerty Learner 的简洁、直接、低干扰体验
- 学习 Type Words / 词文记 的功能深度、自定义词库和数据设计
- 保持 IELTS Typing Learner 自己的垂直定位：雅思机考场景下的自建生词库手打练习工具

当前判断：不应该继续扩大竞品范围。真正值得重点对标的是 Qwerty Learner 和 Type Words / 词文记。

---

## 2. 对比对象

| 产品 | 链接 | 调研定位 |
|---|---|---|
| Qwerty Learner | https://qwerty.kaiyi.cool/ | 简洁体验、输入练习、即时反馈 |
| Type Words / 词文记 | https://typewords.cc/ | 自定义词库、设置体系、功能深度 |
| IELTS Typing Learner | 当前项目 | 雅思机考自建生词库手打练习 |

---

## 3. 总体结论

Qwerty Learner 的优势是“轻”：页面简洁、入口直接、练习体验清楚，用户打开后能快速开始。

Type Words / 词文记 的优势是“深”：功能更完整，有自定义词典、设置、复习机制、统计和更完整的数据能力。

IELTS Typing Learner 不应该简单复制任何一个，而应该形成自己的组合：

> 界面和练习体验学习 Qwerty Learner，数据能力和功能深度参考 Type Words，但产品定位只服务“雅思机考 + 自建生词库 + 手打练习 + 错词复习”。

---

## 4. Qwerty Learner 分析

### 4.1 第一感觉

页面非常简洁，打开后很快能理解这是一个“打字背单词”的工具。整体信息密度不高，练习入口清楚，用户不容易被复杂功能干扰。

### 4.2 值得学习的地方

#### 1. 页面简洁

Qwerty Learner 的第一屏和练习页都比较克制，没有太多解释，也没有把复杂功能全部堆出来。

IELTS Typing Learner 可以学习这一点：  
首页不要做成管理后台，而应该让用户马上知道下一步可以做什么。

#### 2. 词典 / 词库选择

Qwerty Learner 上方可以选择词典。对应到 IELTS Typing Learner，可以理解为“选择词库”。

可以参考的方向：

- 当前词库应该始终清楚显示
- 用户可以快速切换词库
- 词库选择不要藏得太深
- 词库切换后练习入口要明确

#### 3. 即时数据反馈

Qwerty Learner 有练习过程中的即时统计，例如输入速度、正确率、进度等反馈。

IELTS Typing Learner 当前主要是在一轮结束后显示统计。后续可以考虑增加轻量即时反馈，但不要让练习页变复杂。

可以先考虑：

- 当前第几个词
- 本轮已完成数量
- 本轮错误次数
- 当前正确 / 错误反馈

不建议第一版就加入过多打字速度指标。

#### 4. 数据统计图

Qwerty Learner 有一些统计图，例如 WPM 趋势、平均用时、正确率、键盘热力图等。

这些图的具体含义还需要后续进一步拆解，但方向值得参考：

- 用户需要看到自己在进步
- 数据反馈可以增强完成感
- 可视化比单纯数字更有激励感

IELTS Typing Learner 后续可以先做更简单的统计：

- 累计单词数
- 今日新增
- 当前错词数
- 已掌握词数
- 最近练习时间
- 本轮正确 / 错误情况

#### 5. 练习页设置入口

Qwerty Learner 的练习页设置比较方便，比如释义显示、深色模式、单词循环等。

IELTS Typing Learner 可以借鉴“练习页内可快速调整设置”的思路，但要谨慎：

- 设置入口要有，但不能太多
- 不要让用户练习时反复折腾设置
- 释义显示 / 答案显示对不自律用户可能削弱训练效果

#### 6. 深色模式

Qwerty Learner 有深色模式开关。这个功能对体验有帮助，但不是当前第一优先级。

可以列为后续功能，不进入下一阶段核心开发。

### 4.3 不适合照搬的地方

#### 1. 公开网页更偏现成词库

Qwerty Learner 更像是围绕已有词库进行练习。当前没有看到面向普通用户的明显“创建个人词库”入口。

IELTS Typing Learner 的核心差异应该是：

> 用户自己从雅思真题、阅读、听力、写作中逐个录入生词，形成属于自己的词库。

#### 2. 错词本较简洁

Qwerty Learner 的错词相关功能可以参考，但不应该完全照搬。  
IELTS Typing Learner 应该在错词复习上更深入，因为这是“个人积累感”的关键。

后续可以加强：

- 错词出现次数
- 连续正确次数
- 错词减少趋势
- 错词移出规则
- 错词练习入口
- 词库维度的错词统计

#### 3. 不要过度强调 WPM

雅思机考确实需要键盘输入，但 IELTS Typing Learner 的重点不是普通打字速度训练，而是：

> 能否把自己遇到的雅思生词准确打出来。

所以 WPM 可以参考，但不应该成为核心指标。

## 5. Type Words / 词文记 分析

### 5.1 第一感觉

Type Words / 词文记功能很全，比 Qwerty Learner 更复杂。  
它更像一个完整的“打字背词系统”，而不是一个极简练习工具。

### 5.2 值得学习的地方

#### 1. 单词录入功能

这是当前最值得重点学习的部分。

Type Words 的单词录入比单纯输入英文和中文更系统，说明它对“用户自定义内容”这件事比较重视。

IELTS Typing Learner 后续可以重点优化：

- 单词录入表单体验
- 英文 / 中文 / 例句的层级
- 输入后的即时预览
- 重复词提醒
- 来源信息
- 保存后的反馈
- 继续录入的顺滑程度

因为 IELTS Typing Learner 的核心就是“用户自己创建词库”，所以录入体验比普通背词产品更重要。

#### 2. 设置体系

Type Words 的设置内容比较多，包括练习、单词、复习曲线等相关设置。

值得参考的方向：

- 设置项分组
- 用户可以调整学习强度
- 复习机制可以配置
- 单词显示和练习行为可以控制

但 IELTS Typing Learner 不应该一开始做太多设置。普通雅思学生不一定愿意理解复杂设置。

第一版应该只保留少量关键设置：

- 例句显示规则
- 练习顺序
- 是否显示提示
- 后续再考虑复习间隔或掌握规则

#### 3. 复习机制 / 遗忘曲线

Type Words 的复习曲线设置值得研究。  
IELTS Typing Learner 后续如果要做“今天该复习什么”，就需要类似的数据设计。

但当前阶段不建议直接实现复杂算法。应该先设计 DATA_MODEL_V2：

- createdAt
- lastPracticedAt
- practiceCount
- wrongRoundCount
- correctCount
- correctStreak
- mastered
- nextReviewAt（后续可选）

#### 4. 功能深度

Type Words 提供了更完整的学习系统感。  
这说明“打字背词”可以不只是一个练习页面，也可以发展成学习工具。

IELTS Typing Learner 后续可以参考它的功能深度，但必须保持边界：

- 不做大而全英语学习平台
- 不做复杂工具箱
- 不做太多内置词库
- 不做完整雅思模考
- 只围绕自建生词库和手打练习展开

### 5.3 不适合照搬的地方

#### 1. 功能入口可能偏多

Type Words 功能更完整，但对普通雅思学生来说，可能会有理解成本。  
IELTS Typing Learner 不应该一开始就让用户面对大量设置和学习概念。

#### 2. 不要过早引入复杂复习算法

遗忘曲线、FSRS、间隔重复等概念有价值，但对第一版来说太早。

更合理的顺序是：

1. 先记录练习数据
2. 再做首页统计
3. 再做简单掌握状态
4. 最后再考虑复习算法

#### 3. 不要削弱“手动积累感”

如果功能变成批量导入大量词库，可能会削弱产品最初的定位：  
用户逐个录入自己遇到的词，形成个人词汇资产。

因此第一版暂时不做批量导入是合理的。

## 6. IELTS Typing Learner 当前项目取舍

### 6.1 应该学习 Qwerty Learner 的部分

- 首页简洁
- 练习入口直接
- 输入体验顺滑
- 过程反馈清楚
- 不让用户一开始面对太多设置
- 用少量视觉反馈制造完成感

### 6.2 应该学习 Type Words 的部分

- 自建词库能力
- 单词录入体验
- 错词 / 生词管理
- 设置项分组
- 学习统计
- 数据结构的长期扩展能力
- 后续可考虑复习记录和掌握状态

### 6.3 不应该照搬的部分

- 不做大而全背词平台
- 不堆大量内置词库
- 不做复杂工具箱
- 不把 WPM 当作核心目标
- 不做完整雅思模考
- 不过早做复杂复习算法
- 不过早做账号、云同步、付费系统
- 不为了功能丰富牺牲首页简洁

说明：

这里的“不早做账号、云同步、付费系统”只适用于当前前端体验复盘或本地原型阶段，目的是避免在核心页面体验还不清楚时过早进入商业化实现。`commercial-v1` 的长期目标仍然包含注册登录、云端保存、会员系统和支付。

---

## 7. 当前差异化定位

IELTS Typing Learner 的差异化不应该是“我也能打字背词”，而应该是：

> 面向雅思机考用户，把自己在备考中遇到的生词，沉淀成个人词库，并通过键盘输入和错词复习练熟。

更具体地说：

- Qwerty Learner 偏已有词库练习
- Type Words 偏完整打字背词系统
- IELTS Typing Learner 偏雅思机考场景下的个人生词积累

---

## 8. 对下一步 DATA_MODEL_V2 的启发

从本次竞品调研可以看出，下一阶段不应该马上堆功能，而应该先设计数据结构。

因为后续想做以下功能，都依赖数据字段：

- 首页累计词汇
- 今日新增
- 当前错词数
- 已掌握词数
- 最近练习时间
- 每个词练习次数
- 每个词答错轮次
- 错词减少趋势
- 简单复习提醒

因此 DATA_MODEL_V2 至少要考虑：

### 8.1 word 字段

- id
- english
- chinese
- example
- source
- createdAt
- updatedAt
- practiceCount
- correctCount
- wrongRoundCount
- correctStreak
- lastPracticedAt
- mastered
- masteredAt

### 8.2 library 字段

- id
- name
- source
- createdAt
- updatedAt
- lastPracticedAt
- words
- wrongWords 或 word 状态字段
- totalPracticeRounds

### 8.3 practiceHistory

后续可考虑记录每一轮练习：

- id
- libraryId
- mode
- startedAt
- endedAt
- totalWords
- correctWords
- wrongAttempts
- wrongWordIds

### 8.4 dailyStats

后续可考虑每日统计：

- date
- addedWords
- practicedWords
- correctWords
- wrongWords
- masteredWords
- practiceRounds

## 9. 下一步建议

### 9.1 先做 DATA_MODEL_V2.md

原因：  
首页统计、积累感、错词趋势、掌握状态都依赖数据结构。先设计数据，再开发功能，避免返工。

### 9.2 再做首页最小统计

第一版可以只做：

- 累计词汇
- 今日新增
- 当前错词
- 已创建词库
- 最近练习

### 9.3 再优化录入页

因为自建词库是产品核心，录入体验应该比普通背词产品更重要。

### 9.4 最后做用户试用

找 3-5 个真实用户验证：

- 是否能看懂首页
- 是否愿意录入自己的词
- 是否能顺利开始练习
- 是否能找到错词复习
- 是否愿意第二天再打开

---

## 10. 最终结论

Qwerty Learner 最值得学的是：  
简洁、直接、练习体验轻。

Type Words 最值得学的是：  
自定义词库、设置体系、复习记录、数据能力。

IELTS Typing Learner 应该坚持的是：  
雅思机考场景、自建个人生词库、手打输入、错词复习、积累感。

下一步不应该继续随机加功能，而应该先做：

> DATA_MODEL_V2.md

因为只有先设计好数据结构，后面的首页统计、掌握状态、错词趋势和用户积累感才有基础。
