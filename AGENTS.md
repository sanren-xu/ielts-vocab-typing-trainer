# AGENTS.md

## 1. 项目身份

项目名称：IELTS Typing Learner

项目目录：

```text
D:\code\ielts-typing-learner
```

IELTS Typing Learner 是一个面向雅思机考考生的个人生词手打练习工具。

核心产品思路：

> 用户把自己在雅思阅读、听力、写作和学习资料中遇到的生词，手动录入个人词库，然后通过键盘输入练习英文拼写，并通过错词复习把词练熟。

本项目不是：

- 大而全背单词平台
- 完整雅思模考系统
- 通用英语学习平台
- AI 写作批改工具
- AI 口语陪练工具

---

## 2. 分支策略

当前仓库有两个重要分支：

### main

`main` 是稳定的本地版。

特点：

- 纯前端
- HTML / CSS / JavaScript
- localStorage 保存数据
- 不依赖后端
- 不依赖数据库
- 不依赖 Node.js 运行项目

`main` 分支不应该引入：

- Node.js 后端
- 数据库
- 用户账号
- 支付系统
- 服务器部署逻辑

### commercial-v1

`commercial-v1` 是国内商业化 Web 版开发分支。

这个分支允许引入：

- Node.js
- Express
- MySQL
- Backend API
- 注册 / 登录
- 云端词库
- 练习记录云端保存
- 会员系统
- 订单系统
- 支付宝支付
- 留言反馈
- 简单管理后台
- 部署和备案文档

商业化版本开发应该在 `commercial-v1` 分支进行，不要直接在 `main` 上做商业化改造。

## 5. 每次只做一个小功能

每次任务只做一个小功能，不要顺手改一堆东西。

可以接受的小任务：

- 新增一个文档
- 修改一个文档
- 创建 backend 骨架
- 增加 `/api/health`
- 增加一个注册接口
- 增加一个登录接口
- 新增一个数据库建表脚本

不允许在一个任务里同时做：

- 重写整个项目
- 一次性开发全部后端
- 一次性重做全部页面
- 同时做登录、支付、会员、后台

## 8. 安全规则

绝对不要提交敏感信息。

不要提交：

- `.env`
- 数据库密码
- MySQL root 密码
- 支付宝私钥
- 支付宝公钥
- 支付证书
- API Key
- 真实用户数据
- 生产数据库备份
- 导出的用户数据

如果需要环境变量文件，只创建：

```text
.env.example
```

不要创建并提交真实 `.env`。

`.env.example` 只能写占位值，例如：

```env
PORT=3000
DB_HOST=localhost
DB_PORT=3306
DB_USER=ielts_app
DB_PASSWORD=your_database_password
DB_NAME=ielts_typing_learner
```

## 10. 数据库规则

商业化版本使用 MySQL。

正式数据不能再只依赖 localStorage。

数据流应该是：

```text
前端页面
→ 后端 API
→ MySQL 数据库
```

不要让前端页面直接连接 MySQL。

不要把 SQL 逻辑散落在 HTML 页面或前端 JS 里。

创建或修改数据库表之前，必须说明：

1. 涉及哪张表
2. 新增或修改哪些字段
3. 是否影响旧数据
4. 是否有回滚方案
5. 如何在 DBeaver 里验收

禁止随便执行危险操作：

```sql
DROP TABLE
TRUNCATE TABLE
DELETE FROM ...
```

除非用户明确确认，并且确认这是本地测试数据。

## 11. API 规则

后端 API 应该保持结构清晰。

基本要求：

1. 路由分组清楚
2. 返回格式统一
3. 校验用户输入
4. 登录后才能访问用户自己的数据
5. 不相信前端传来的会员状态
6. 不相信前端传来的支付状态
7. 价格、会员、订单必须以后端和数据库为准
8. 新增 API 后要给 Apifox 测试方法

建议成功返回格式：

```json
{
  "ok": true,
  "data": {},
  "message": "success"
}
```

建议失败返回格式：

```json
{
  "ok": false,
  "message": "error message"
}
```

## 12. 支付规则

支付逻辑必须在后端处理。

不能因为前端页面显示“支付成功”就开通会员。

支付宝支付流程必须是：

1. 用户创建订单
2. 后端生成支付请求
3. 用户完成支付
4. 支付宝发送异步通知
5. 后端验签
6. 后端核对订单号
7. 后端核对金额
8. 后端确认支付状态
9. 订单状态改为 `paid`
10. 自动开通或延长会员
11. 保存支付记录

支付回调必须做到幂等。

也就是说：

> 支付平台重复通知时，不能重复开通会员或重复加时间。
