# FRONTEND_REVIEW.md

## 1. 文件用途

本文件用于记录 IELTS Typing Learner 的前端页面复盘和商业化版本页面改造计划。

当前用户已经明确反馈：

对现在的页面已有功能不满意，商业化版本不能直接把旧页面硬接后端。

因此，在继续写后端、数据库和支付之前，需要先复盘前端页面。

本文件主要回答：

1. 当前有哪些页面
2. 每个页面现在负责什么
3. 当前页面有哪些问题
4. 商业化版本需要哪些页面
5. 哪些页面保留
6. 哪些页面重做
7. 哪些页面新增
8. 哪些页面先做静态原型
9. 哪些页面后续接 API

## 4. 商业化版本前端总原则

商业化版本不能简单地把当前页面接上后端。

原因：

1. 当前页面主要为本地 localStorage 设计
2. 商业化版本需要注册 / 登录
3. 商业化版本需要会员状态
4. 商业化版本需要免费额度
5. 商业化版本需要订单和支付入口
6. 商业化版本需要用户反馈
7. 商业化版本需要管理后台
8. 用户第一次访问时需要更清楚的引导

商业化前端改造原则：

1. 先复盘页面，不急着接 API
2. 先确定页面结构，再写后端接口
3. 页面数据来源要明确：localStorage、API，还是临时 mock data
4. 不盲目引入 React / Vue
5. 优先复用现有 `style.css`
6. 不要重写整个项目
7. 每次只改一个页面或一个小区域
8. 每次改完都要说明测试方式

## 5. index.html 页面复盘

### 当前定位

`index.html` 当前是首页入口。

主要作用：

1. 进入录入页
2. 进入词库页
3. 进入练习页
4. 进入错词页
5. 进入设置页

### 当前可能存在的问题

1. 更像功能入口，不像学习首页
2. 缺少学习进度反馈
3. 缺少当前词库状态
4. 缺少今日新增词数量
5. 缺少当前错词数量
6. 缺少最近练习时间
7. 对新用户来说，不一定知道第一步该做什么
8. 商业化版本缺少登录状态和会员状态展示

### 商业化版本可能需要承担的功能

1. 登录后学习首页
2. 当前词库概览
3. 今日新增单词
4. 当前错词数量
5. 累计录入单词
6. 最近练习时间
7. 学习连续天数
8. 免费额度使用情况
9. 会员状态
10. 快速开始练习
11. 快速录入单词

### 建议方向

商业化版本中，首页应该从“导航入口”升级为“学习仪表盘”。

建议新增最小统计：

1. 当前词库数
2. 当前单词数
3. 当前错词数
4. 最近练习时间
5. 今日新增词数

6. 容易偏离产品定位

3. 后续作为高级导入功能
4. 删除或隐藏入口
5. 合并到 `input.html`

V1 暂时不恢复批量导入。

先做好手动录入体验。

## 16. 前端和后端的配合原则

不能简单按照：

前端全部做完  
→ 再写后端  
→ 再设计数据库

更合理的顺序是：

1. 先复盘前端
2. 明确页面结构
3. 明确每个页面需要的数据
4. 设计 API
5. 设计数据库
6. 写后端
7. 前端先用 mock data 做静态原型
8. 再逐步接真实 API
